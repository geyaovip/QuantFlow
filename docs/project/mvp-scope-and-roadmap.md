# QuantFlow MVP 范围与路线

状态：基线规格｜目标版本：`v1.0.0 MVP`

## 1. 产品闭环

1. **官网转化**：用户在 `/` 理解品牌定位、核心功能、策略信号、模拟盘、风控理念、定价和风险提示。
2. **进入工作台**：用户点击官网“进入应用”，未登录先进入 `/login?next=/app/strategies`，登录成功后进入 `/app/strategies`。
3. **发现**：在应用工作台浏览、筛选和比较带风险信息的策略。
4. **理解**：查看信号、策略逻辑、失效条件和数据来源。
5. **验证**：创建模拟盘，观察模拟权益、回撤、交易和风险事件。
6. **订阅**：按会员权益获得数据范围、提醒能力和模拟盘容量；不销售收益预期。
7. **治理**：管理员在 `/admin` 完成内容审核、风控、权限和审计闭环。

## 2. 版本范围

### P0：`v1.0.0` 必须完成

| 域         | 能力                                                                                                        |
| ---------- | ----------------------------------------------------------------------------------------------------------- |
| 官网       | 根路径 `/` 官网首页、产品定位、核心功能、策略信号预览、模拟盘说明、风控理念、定价、风险提示、“进入应用” CTA |
| 应用工作台 | `/app` 基础布局，必须重定向到 `/app/strategies`                                                             |
| 账号       | 用户端/管理端 Resend 邮箱验证码登录、首次登录建号、退出、个人资料、安全记录                                 |
| 策略       | 广场、筛选、详情、指标、订阅、平台端创建与审核                                                              |
| 信号       | 列表、详情、访问控制、过期/取消、提醒、加入模拟盘                                                           |
| 模拟盘     | 创建、列表、详情、暂停/恢复、模拟订单/成交/持仓、权益与回撤、风险事件                                       |
| 会员       | Free/Pro/Premium 权益模型、订阅生命周期、服务端配额校验、人工/测试开通                                      |
| 通知       | 站内通知、风险提醒、系统公告、通知偏好                                                                      |
| 管理端     | `/admin` 数据看板、用户、策略、信号、模拟盘、会员、风险、角色权限、审计日志                                 |
| 平台       | 中文文案、风险披露、feature flag、可观测错误、测试和发布门禁                                                |

`v1.0.0` 允许通过 Plisio 开通 Pro/Premium 会员；支付只购买功能容量，不承诺策略收益，不接入交易所连接、真实下单、自动续费或自动退款。

### P1：不阻断 `v1.0.0`

风险测评、策略收藏/对比、高级筛选、邮件通知、行情中心、帮助中心、后台内容管理、高级报告。

### P2：MVP 禁止

交易所连接、API Key、真实资产/订单/持仓、半自动或全自动交易、作者自助入驻、返佣分销、原生 App。

## 3. 交付里程碑

| 版本     | 可演示结果                                                         | 准入重点                                   |
| -------- | ------------------------------------------------------------------ | ------------------------------------------ |
| `v0.1`   | 文档基线与决策边界                                                 | 文档链接、范围和术语一致                   |
| `v0.2`   | 可启动的前后端、数据库、CI、Auth 骨架                              | 技术基线、Resend OTP 与禁用 flag 生效      |
| `v0.3`   | 官网 `/` → “进入应用” → 登录 → `/app/strategies` → 策略详情 → 信号 | 三端路由边界、收益风险配对、审核和访问控制 |
| `v0.4`   | 信号 → 模拟盘 → 权益/回撤                                          | 撮合公式版本化、无实盘链路                 |
| `v0.5`   | 会员权益、通知和 `/admin` 管理治理                                 | 配额后端校验、RBAC、审计日志               |
| `v0.6`   | 发布候选版                                                         | E2E、安全、性能、合规扫描、回滚演练        |
| `v1.0.0` | MVP 正式版                                                         | `acceptance-criteria.md` 全部 P0 门禁通过  |

## 4. 成功指标

MVP 上线后连续 30 天使用以下初始目标：策略列表会话到详情到达率 `>=25%`；注册用户 7 天内创建模拟盘比例 `>=15%`；模拟盘 30 日留存 `>=20%`；订阅策略用户周信号查看率 `>=40%`；关键风险确认记录完整率 `100%`；模拟计算失败率 `<0.5%`；stale 行情快照占比 `<1%`；critical 风险事件 30 分钟内响应率 `>=95%`；用户投诉人数占 MAU `<1%`；会员支付成功回调开通完整率 `100%`。

真实交易量、真实资产规模和承诺收益不属于 MVP 成功指标。

## 5. 范围变更

任何新增 P0/P1/P2 或版本移动都必须同时更新本文件、功能拆解、验收标准和决策日志。涉及实盘能力的请求必须停止实现并先重新评审产品、安全、风控与合规边界。
