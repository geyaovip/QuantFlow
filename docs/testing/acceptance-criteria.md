## 1. 文档目标

本文档定义 QuantFlow MVP 的测试验收标准，用于产品验收、设计走查、前端测试、后端测试、接口测试、安全测试和上线准入。

QuantFlow MVP 不包含真实交易所 API 连接、真实资产读取、真实下单、半自动交易和全自动交易。测试过程中若发现以上入口或可执行接口，应视为阻断级问题。

## 2. 全局验收标准

1. 项目名称统一为 QuantFlow。
2. 不使用中文品牌名。
3. 所有收益展示都必须同时展示风险指标。
4. 所有风险提示必须明确说明不承诺收益。
5. 所有策略历史表现都必须提示历史表现不代表未来收益。
6. 所有模拟盘页面必须明确标识模拟，不得与真实交易混淆。
7. UI 风格专业、克制、金融科技感，不得像资金盘或暴富项目。
8. MVP 不展示真实交易所连接、真实下单、半自动交易和全自动交易入口。
9. MVP 用户可见页面、按钮、表单、错误提示、风险提示、通知和管理后台操作文案默认使用简体中文。
10. 英文仅用于 QuantFlow 品牌名、币种符号、技术缩写、API 字段、代码标识和行业通用术语。
11. 用户端和管理端均使用 Resend 邮箱验证码登录，不存在密码注册、密码登录或密码重置入口。
12. MVP 默认浅色主题；页面以白/灰/黑为主，主按钮深灰/黑，低饱和蓝只用于链接、选中态和信息提示。
13. 产品结构采用 `/` 官网营销页、`/app` 应用工作台、`/admin` 管理后台三端分离。

## 3. 用户端验收标准

### 3.1 官网首页 `/`

验收标准：

1. 首屏能说明 QuantFlow 提供策略信号、模拟盘和风险控制。
2. 首屏或页面关键区域说明 MVP 不需要连接交易所。
3. 热门策略模块同时展示近 30 日表现和最大回撤。
4. 实时信号预览不提供真实下单入口。
5. 首页底部展示标准免责声明。
6. 右上角提供“进入应用”主 CTA，点击进入 `/app/strategies`。
7. 官网负责品牌、信任、转化和风险教育，不承载完整应用功能或管理操作。
8. 官网不出现稳赚、保本、躺赚、零风险等禁止表达。

### 3.2 应用工作台 `/app`

验收标准：

1. `/app` 必须重定向到 `/app/strategies`。
2. 应用端承载策略广场、策略详情、信号中心、模拟盘、我的策略、会员订阅和个人中心。
3. 应用端更强调数据、策略、信号、模拟盘和风险并列展示。
4. 应用端不展示真实交易所连接、真实下单、半自动交易或全自动交易入口。

### 3.3 策略广场

验收标准：

1. 支持按币种、策略类型、风险等级、收益周期、最大回撤、是否免费、是否支持模拟盘筛选。
2. 支持综合推荐、最新上线、订阅人数、近 7 日表现、近 30 日表现、最大回撤、风险等级排序。
3. 策略卡片展示收益、最大回撤、风险等级、胜率和信号频率。
4. 已暂停、风险观察、已下架策略状态处理正确。
5. 不出现收益承诺或禁止表达。

### 3.4 策略详情

验收标准：

1. 展示策略头部、当前信号、表现概览、收益曲线、回撤曲线、历史信号、策略逻辑、风险说明。
2. 收益曲线与回撤曲线相邻展示。
3. 回测数据和模拟数据明确区分。
4. 数据样本不足时展示提示。
5. 操作按钮仅包含订阅、开始模拟、查看信号、添加提醒等 MVP 操作。
6. 不展示半自动执行或开启自动交易按钮。

### 3.5 信号中心

验收标准：

1. 支持全部、买入、卖出、观望、有效、过期、已模拟、已收藏分类。
2. 信号字段完整，包括方向、触发价格、当前价格、建议仓位、止损、止盈、有效期、风险等级。
3. 信号详情展示触发原因、策略说明、历史相似信号表现和风险提示。
4. 过期、取消、策略暂停、风控观察信号不可加入新的模拟执行。
5. 不存在真实下单入口。

### 3.6 模拟盘

验收标准：

1. 创建模拟盘需选择策略、币种、初始资金、仓位、止损止盈规则。
2. 默认初始资金为 10,000 USDT。
3. 默认单笔最大仓位为 10%。
4. MVP 杠杆固定为 1 倍或不展示杠杆。
5. 创建前明确提示仅为模拟验证，不承诺收益。
6. 详情页展示当前权益、累计收益、最大回撤、持仓、历史交易、收益曲线、回撤曲线、风控触发记录。
7. 暂停、恢复、删除、复制操作状态正确。

### 3.7 会员订阅

验收标准：

1. 展示 Free、Pro、Premium 权益，且数值来自统一配置。
2. 权益描述围绕策略访问、实时信号、模拟盘数量、历史数据、通知渠道和风险分析。
3. 不使用收益承诺作为会员申请或开通理由。
4. 申请或开通会员前展示标准免责声明。
5. 人工、邀请码和测试订阅状态正确；页面无在线购买、支付结果、自动续费或退款入口。

## 4. 管理端验收标准

管理端路径为 `/admin`，独立承载运营管理功能。管理端更强调高密度表格、筛选、状态管理和操作效率。

### 4.1 数据看板

1. 展示用户、策略、信号、模拟盘、商业和风险数据。
2. 增长指标旁边包含风险或异常指标。
3. 数据延迟有说明。

### 4.2 策略管理

1. 策略支持草稿、待审核、已上线、已暂停、风险观察、已下架状态。
2. 策略上线前必须完成审核项。
3. 审核必须检查收益承诺、风险说明、最大回撤、适合行情、不适合行情和样本量。
4. 上线、暂停、下架记录审计日志。

### 4.3 信号管理

1. 可查看信号列表和详情。
2. 可取消信号、标记异常、重新推送。
3. 异常信号生成风险事件。
4. 操作记录审计日志。

### 4.4 模拟盘管理

1. 可查看全部用户模拟盘。
2. 可暂停、恢复、标记异常。
3. 可查看模拟交易记录和风控事件。
4. 不得将模拟盘标识为真实交易。

### 4.5 会员管理

1. 可分页查看订阅列表和详情。
2. 可人工开通、延长和取消，不存在订单或退款操作。
3. 可手动调整会员且必须记录审计日志。
4. MVP 不存在财务角色、订单或退款权限。

### 4.6 权限管理

1. 管理端 RBAC 生效。
2. 权限不足时接口返回 403。
3. 权限变更记录审计日志。

## 5. 数据与接口验收标准

1. 所有列表接口使用服务端分页并返回 page/pageSize/total/totalPages；用户默认 20、管理端默认 50、最大 100。
2. pageSize 只接受用户端 10/20/50 或管理端 20/50/100；筛选、搜索和排序变化后回到第 1 页。
3. 所有分页有稳定排序；最后一页、空列表、超范围页和并发新增数据场景有测试，MVP 无无限滚动。
4. 策略列表和详情返回收益指标时同时返回风险指标。
5. 信号详情返回风险提示和免责声明。
6. 模拟盘接口不产生真实订单。
7. 后续交易相关接口在 MVP 返回功能未开放或不可访问。
8. MVP 未注册支付创建、支付查询和支付 webhook API。
9. 管理端敏感操作接口强制 RBAC。
10. 所有金额、价格、比例精度正确。

## 6. 风险与合规验收标准

1. 首页、策略详情、信号详情、订阅前、创建模拟盘前、会员申请/开通页均展示免责声明。
2. 禁止表达扫描通过，包括稳赚、保本、固定收益、零风险、必涨、百分百胜率、躺赚、无脑赚钱、永不爆仓、稳赚不亏。
3. 高风险策略展示风险等级和原因。
4. 数据样本不足时展示提示。
5. 风险事件可以从触发到处理闭环追踪。

## 7. 非功能验收标准

### 7.1 性能

1. 官网首页加载时间小于 3 秒。
2. 策略列表加载时间小于 3 秒。
3. 信号生成后推送延迟尽量控制在 10 秒内。
4. 模拟盘数据更新延迟控制在 1 分钟内。
5. 管理端统计数据允许 5-15 分钟延迟。

### 7.2 安全

1. 验证码仅存 hash，单次使用并校验有效期、重发失效、最大尝试次数和并发验证。
2. 支持 HTTPS。
3. 防 SQL 注入。
4. 防 XSS。
5. 防 CSRF。
6. 管理端权限控制生效。
7. 管理员操作记录日志。
8. MVP 不接收真实交易所 API Key。
9. Resend API Key、验证码和会话秘密不出现在客户端、日志或错误响应中。
10. 用户与管理员会话隔离，未知管理员不能因验证邮箱而获得管理端账号。
11. 生产 OTP 请求必须通过 Turnstile 服务端校验；缺失、伪造、过期和重复 token 均不发送邮件，且无法绕过后端 IP + 邮箱限流。
12. Web、Admin 和 API 只通过 Cloudflare Tunnel 公开；应用端口仅绑定 loopback，PostgreSQL 和 Worker 无公网端口。

### 7.3 稳定性

1. 行情数据异常时有告警。
2. 模拟盘计算异常时可暂停并标记。
3. 异步任务失败有重试和告警。
4. 关键状态变更可追踪。

### 7.4 可维护性

1. 单个源码文件默认不超过 300 行，超过 400 行必须拆分或属于明确例外。
2. React 组件默认不超过 200 行，React 页面默认不超过 250 行。
3. 后端 Controller 默认不超过 200 行，Service 默认不超过 300 行。
4. 单个函数默认不超过 60 行。
5. 页面、组件、hook、API、权限、格式化、业务规则分层清晰。
6. 后端采用轻量 DDD + 模块化单体，按业务领域拆分模块。
7. 核心业务模块具备 domain、application、infrastructure、interfaces 边界；简单模块未过度抽象。
8. Controller、Use Case、Domain、Repository、Adapter 边界清晰。
9. Controller 不包含交易、策略、模拟盘或风控核心逻辑。
10. MVP 未引入微服务、复杂 CQRS、Event Sourcing 或复杂事件总线。
11. 模拟交易命名与后续真实交易预留命名没有混淆。
12. 禁止魔法字符串散落表示策略状态、信号状态、会员等级和风险等级。

### 7.5 可访问性

1. 核心路径可仅使用键盘完成，焦点顺序和焦点样式清晰。
2. 表单控件有可访问名称，错误与字段关联，异步状态可被辅助技术识别。
3. 颜色不是风险、涨跌和状态的唯一表达；关键文本满足 WCAG 2.2 AA 对比度。
4. 收益和回撤图表提供文本摘要或可访问数据表。
5. Button、表单、Dialog 和表格操作均覆盖 focus-visible、hover、disabled；Dialog 正确管理焦点。

### 7.6 视觉一致性

1. 官网为轻量营销表达，但仍专业、克制、可信，“进入应用”是主要 CTA。
2. 应用工作台为浅灰背景、白色卡片、轻边框和少阴影；不存在霓虹、重投影或高饱和渐变。
3. 管理端为侧边栏、白色内容区和高密度表格，数值与操作列对齐稳定。
4. 无深色主题切换入口；未来深色 token 不进入 MVP 视觉回归矩阵。
5. 蓝色使用频率低于中性色和深色主操作，不承担大面积品牌背景。
6. 收益、亏损和警示色只表达数据语义，且颜色不是唯一状态提示。
7. v0 颜色 token 与 `ui-guidelines.md` 的名称和值一致，页面和组件不散落重复十六进制主题色。
8. `Muted Text`、`Info`、`Profit`、`Warning` 在白底小号文本场景遵守使用限制；不得仅因 token 已定义就跳过实际对比度检测。
9. 字体、字号、间距、圆角、阴影、控件高度、焦点样式和动效时长符合 v0 tokens，没有页面级替代体系。

### 7.7 自适应布局

1. 官网、应用工作台和管理端核心页面至少在窄屏、中等宽度和宽屏三类视口完成布局与交互测试。
2. 卡片、表单、图表、Dialog、侧边栏和 DataTable 不出现遮挡、不可达操作或不可恢复的横向溢出。
3. 官网移动端可以使用顶部导航 + 汉堡菜单，且“进入应用”仍清晰可见。
4. `/app` 移动端必须使用固定底部导航栏作为主导航，不使用左上角汉堡菜单作为主导航。
5. `/app` 顶部区域只承担页面标题、返回、通知、搜索等辅助操作，左上角最多用于 Logo 或返回按钮。
6. `/app` 底部导航触控目标可点击、适配安全区，不遮挡页面主操作按钮、表单提交区、风险提示或 Toast。
7. 管理端以桌面端为主且不要求底部导航，但移动端仍可完成权限允许的必要查看和管理操作。
8. DataTable 可通过横向滚动或详情视图访问完整信息，关键风险、状态和权限信息不被隐藏。
9. 自适应变化不导致收益与配对风险分离，也不弱化免责声明。
10. 断点覆盖 640、768、1024、1280、1536px 临界值前后，避免只测固定设备截图。

## 8. MVP 上线准入

阻断级问题：

1. 出现真实交易所连接入口。
2. 出现真实下单入口。
3. 出现半自动交易或全自动交易入口。
4. 出现收益承诺或禁止表达。
5. 策略收益无风险指标。
6. 模拟盘被误标识为真实交易。
7. 管理端权限绕过。
8. 支付或会员权益校验错误。
9. 风险免责声明缺失。
10. 核心业务文件明显超出代码组织阈值且未拆分。
11. `/`、`/app`、`/admin` 路径职责混淆，或官网 CTA 无法进入应用工作台。
12. `/app` 移动端使用左上角汉堡菜单作为主导航，或底部导航遮挡主操作按钮/风险提示。
13. Controller 中写入策略、信号、模拟盘、风控或交易执行核心逻辑。

上线前必须完成：

1. P0 功能验收。
2. 核心接口测试。
3. 权限测试。
4. 风险提示走查。
5. 策略数据校验。
6. 模拟盘计算校验。
7. 管理端审计日志校验。
8. 基础安全测试。
9. 代码组织和可维护性走查。
