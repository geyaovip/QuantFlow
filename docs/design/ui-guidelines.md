# QuantFlow UI 设计规范

状态：MVP v0 设计系统已确定｜后续调整必须版本化

## 1. 视觉定位

QuantFlow MVP 默认使用浅色主题，呈现专业、克制、冷静、可信的数据平台感。视觉层级可参考 [Glassnode Studio](https://studio.glassnode.com/) 的信息密度与数据产品气质，但不得复制其布局、组件、品牌元素或视觉细节。

避免资金盘、博彩盘、交易喊单、暴富项目、霓虹加密风和夸张收益营销。默认简体中文，品牌名只使用 QuantFlow。

品牌母版、浏览器图标、PWA 图标、用户端与管理端组合标识以 [`assets/brand/README.md`](../../assets/brand/README.md) 为唯一资源入口。实现不得自行重绘、裁切或从截图提取 Logo。

## 2. 主题策略

1. MVP 只要求浅色主题完整、可访问、经过视觉测试。
2. 深色主题不是 MVP 功能，不提供主题切换入口，不为深色模式维护独立页面分支。
3. 颜色、间距、圆角、阴影和排版必须通过语义化 design tokens / CSS variables 使用，避免页面硬编码。
4. 可以为未来深色主题保留 token 覆盖层结构，但不得因此增加第一版开发和测试范围。

## 3. 色彩体系

以下为可直接实现的 `v0` 语义色 tokens。`v0` 表示 MVP 初始基线，后续允许通过版本化变更调整；组件只能引用 token 名称，不依赖具体十六进制值。

| Token               | CSS Variable                  | v0 Value  | 用途                     |
| ------------------- | ----------------------------- | --------- | ------------------------ |
| Background          | `--color-background`          | `#F7F8FA` | 页面画布                 |
| Surface             | `--color-surface`             | `#FFFFFF` | 卡片、表格、弹窗         |
| Surface Muted       | `--color-surface-muted`       | `#F3F4F6` | 次级区域、表头、禁用背景 |
| Border              | `--color-border`              | `#E5E7EB` | 边框与分隔线             |
| Primary Text        | `--color-text-primary`        | `#111827` | 标题、正文重点           |
| Secondary Text      | `--color-text-secondary`      | `#4B5563` | 正文与辅助说明           |
| Muted Text          | `--color-text-muted`          | `#9CA3AF` | 低优先级元信息           |
| Primary Action      | `--color-action-primary`      | `#111827` | 主要操作背景             |
| Primary Action Text | `--color-action-primary-text` | `#FFFFFF` | 主要操作文字             |
| Accent              | `--color-accent`              | `#64748B` | 中性强调、次级选中       |
| Info                | `--color-info`                | `#0284C7` | 链接、信息和低频选中态   |
| Profit              | `--color-profit`              | `#16A34A` | 收益/正向数据            |
| Loss                | `--color-loss`                | `#DC2626` | 亏损/危险数据            |
| Warning             | `--color-warning`             | `#D97706` | 警示状态                 |

token 变更必须更新版本记录、组件视觉回归和对比度检查。不得从这些基础色直接推导收益承诺或营销装饰。

### 3.1 v0 token 的可访问性限制

这些值是基础语义色，不代表任意前景/背景组合都满足 WCAG 2.2 AA。以 `Surface #FFFFFF` 为背景时：

- `Primary Text`、`Secondary Text`、`Accent`、`Loss` 可满足普通文本 4.5:1 基线。
- `Muted Text` 约 2.54:1，只能用于禁用态或非必要装饰信息，不能用于正文、表单提示、placeholder、风险、时间和关键元信息。
- `Info` 约 4.10:1，不能作为白底小号链接文字的唯一前景色；链接必须增加下划线，并使用通过实际字号/字重检测的可访问前景色。
- `Profit` 约 3.30:1、`Warning` 约 3.19:1，不能单独用于白底普通文本；小号数据文字使用 `Primary Text`，再配合语义图标、边框或背景，或后续新增经过验证的 foreground token。

可访问性优先于严格复用基础色。任何新增 foreground/on-color token 都必须记录对比度、背景和使用范围。

### 3.2 中性色为主体

- 页面背景：浅灰，用于区分应用画布和内容表面。
- 内容表面：白色，用于卡片、表格、弹窗和表单。
- 主文字：接近黑色；次级文字使用中性灰。
- 边框与分隔：浅灰细线，优先使用边框建立层级。
- 主操作：深灰或黑色背景、白色文字。
- 次操作：白色或透明背景、深色文字、轻边框。

不得使用大面积高饱和蓝色作为品牌背景、卡片底色或主要装饰。

### 3.3 辅助蓝

低饱和蓝灰只用于链接、当前选中态、信息提示、轻量焦点状态和少量图表辅助序列。蓝色不能与主按钮争夺视觉中心，也不能代替风险语义色。

### 3.4 数据语义色

- 收益/正向：克制的绿色。
- 亏损/高风险：克制的红色。
- 警示：琥珀色。
- 中性/无变化：灰色。

语义色只表达事实状态，不作为品牌色或营销装饰。颜色不是唯一信息渠道，必须配合文字、图标或形状；收益与亏损视觉权重保持平衡。

## 4. 表面、边框与阴影

1. 用户端使用浅灰页面背景、白色卡片、轻边框和极少阴影。
2. 优先通过背景、边框和留白建立层级，阴影只用于浮层或必要的层级分离。
3. 禁止高强度投影、玻璃拟态、发光描边、高饱和渐变和装饰性行情纹理。
4. 圆角保持克制且全局一致，使用下表 v0 tokens。
5. 页面先保证结构清晰、信息完整和风格统一，不以最终高保真为 MVP 完成条件。
6. 不为视觉效果引入复杂动效、强渐变、大面积高饱和色块或高成本装饰层。

### 4.1 圆角、边框和阴影 tokens

| Token          | CSS Variable       | v0 Value                             | 用途                    |
| -------------- | ------------------ | ------------------------------------ | ----------------------- |
| Radius Small   | `--radius-sm`      | `4px`                                | 小标签、紧凑控件        |
| Radius Medium  | `--radius-md`      | `8px`                                | Button、Input、Select   |
| Radius Large   | `--radius-lg`      | `12px`                               | Card、Dialog、Drawer    |
| Radius Full    | `--radius-full`    | `9999px`                             | Badge、圆形控件         |
| Border Width   | `--border-width`   | `1px`                                | 默认边框                |
| Shadow Small   | `--shadow-sm`      | `0 1px 2px rgba(17, 24, 39, 0.04)`   | 必要的轻层级            |
| Shadow Overlay | `--shadow-overlay` | `0 12px 32px rgba(17, 24, 39, 0.12)` | Dialog、Popover、Drawer |

普通 Card 默认 `1px Border + 无阴影`；只有悬浮交互卡或需要与背景分离时使用 `Shadow Small`。禁止新增中/重卡片阴影。

## 5. 间距与尺寸

使用 4px 基准间距，不使用未进入 token 的任意值。

| Token | CSS Variable | Value  |
| ----- | ------------ | ------ |
| 0     | `--space-0`  | `0`    |
| 1     | `--space-1`  | `4px`  |
| 2     | `--space-2`  | `8px`  |
| 3     | `--space-3`  | `12px` |
| 4     | `--space-4`  | `16px` |
| 5     | `--space-5`  | `20px` |
| 6     | `--space-6`  | `24px` |
| 8     | `--space-8`  | `32px` |
| 10    | `--space-10` | `40px` |
| 12    | `--space-12` | `48px` |
| 16    | `--space-16` | `64px` |

- 默认控件高度 `40px`，紧凑管理端控件 `32px`，移动端主要触控控件不低于 `44px`。
- 用户端 Card padding：窄屏 `16px`，中宽及以上 `20px` 或 `24px`。
- 管理端表头高度 `40px`、默认数据行 `44px`；超高密度 `36px` 仅用于只读且无复杂操作的表格。
- 页面模块间距窄屏 `24px`、宽屏 `32px`；同组字段间距 `16px`。

## 6. 排版与数据

字体栈使用系统字体，避免 MVP 增加字体网络依赖：

```text
ui-sans-serif, -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", Arial, sans-serif
```

| Token         | Size / Line Height | 典型用途                 |
| ------------- | ------------------ | ------------------------ |
| `--text-xs`   | `12px / 18px`      | Badge、非关键元信息      |
| `--text-sm`   | `14px / 20px`      | 表格、表单帮助、辅助文本 |
| `--text-base` | `16px / 24px`      | 正文、默认输入           |
| `--text-lg`   | `18px / 28px`      | 卡片标题                 |
| `--text-xl`   | `20px / 28px`      | 模块标题                 |
| `--text-2xl`  | `24px / 32px`      | 页面标题（窄屏）         |
| `--text-3xl`  | `30px / 38px`      | 页面标题（宽屏）         |
| `--text-4xl`  | `36px / 44px`      | 仅首页关键标题           |

字重仅使用 400、500、600；正文 400，控件/表头 500，标题 600。避免大面积粗体。

1. 中文、英文和数字共用上述兼容字体栈。
2. 数字启用 tabular numerals，金额、百分比和时间列对齐。
3. 页面标题、模块标题、正文和辅助信息使用有限且稳定的层级，不依赖超大字号制造视觉冲击。
4. 风险提示不得小于正文可读下限，免责声明不能被弱化成装饰性脚注。

## 7. 官网布局

1. 官网路径为 `/`，负责品牌、信任、转化和风险教育。
2. 官网可以比应用端更轻量、更有营销表达，但必须保持专业、克制、可信。
3. 官网首屏包含清晰产品定位、风险友好的副标题和“进入应用”主 CTA。
4. 官网右上角固定或显著展示“进入应用”，未登录时点击进入 `/login?next=/app/strategies`。
5. 官网模块可使用更大的标题、更多留白和更少表格，但不得使用金币、火箭、收益爆炸、霓虹加密风或暴富暗示。
6. 官网展示策略信号、模拟盘和定价时，必须同时展示风险提示或“不承诺收益”说明。
7. 官网不承载完整应用操作，不在官网内完成模拟盘管理、策略深度筛选或后台操作。

## 8. 应用工作台布局

1. 应用工作台路径前缀为 `/app`，负责具体功能使用。
2. 浅灰应用背景，核心内容放在白色卡片或白色连续表面中。
3. 策略列表采用响应式卡片/表格组合；关键筛选保持易发现。
4. 策略卡必须同时展示至少一个收益指标和一个同周期风险指标。
5. 详情页将收益、最大回撤、样本量、数据来源和更新时间组织在相邻区域。
6. 风险披露靠近订阅、创建模拟盘等关键操作，不只放在页脚。
7. 应用端更强调数据、策略、信号、模拟盘和风险并列展示，信息密度高于官网。
8. 应用工作台移动端必须使用底部导航栏作为主导航，不使用左上角汉堡菜单作为主导航。
9. 应用工作台顶部区域用于当前页面标题、返回、通知、搜索等辅助操作；左上角不作为主功能入口，最多用于 Logo 或返回按钮。
10. 底部导航保持简洁、固定、易点击，适配安全区，不得遮挡页面主操作按钮或风险提示。

## 9. 管理端布局

1. 管理端路径前缀为 `/admin`，负责运营管理。
2. 使用侧边栏 + 白色内容区的桌面管理结构。
3. 以高密度表格、筛选栏、状态标签和详情抽屉为核心，减少无意义大卡片。
4. 数值列右对齐，操作列稳定，批量操作和导出受权限控制。
5. 危险操作二次确认；审计日志入口靠近敏感操作。
6. 高密度不等于拥挤：表头、行高、空态、错误态和分页保持清晰。
7. 侧边栏宽 `240px`，折叠宽 `72px`；顶部栏高 `64px`，主内容区使用流式宽度。
8. 管理端以桌面端为主，移动端不是核心场景，但仍须支持权限允许的必要查看和管理操作。

## 10. 数据可视化

1. 收益率与同周期最大回撤并列，胜率与交易次数、盈亏比并列。
2. 收益曲线与回撤曲线相邻；回测、模拟和行情来源使用明确图例区分。
3. 图表以中性灰为基础，语义色和低饱和蓝作为有限序列色，不使用大面积品牌蓝。
4. tooltip 必须展示时间、值、数据来源和必要风险信息。
5. 加载失败、数据延迟、样本不足和空数据均有明确文本状态。
6. 图表提供文本摘要或可访问数据表。

## 11. 交互与状态

- Primary：深灰/黑色主按钮，每个区域通常只有一个主操作。
- Secondary：白色或透明底、轻边框。
- Link / Selected / Info：低饱和蓝。
- Danger：仅用于真实危险操作，不用于吸引点击。
- Disabled：同时降低对比并解释不可用原因。

所有组件必须覆盖 default、hover、focus-visible、active、disabled、loading、error 状态。核心路径支持键盘操作，表单有语义标签和关联错误，正文和关键指标满足 WCAG 2.2 AA 对比度。

- focus-visible：`2px solid #0284C7`，offset `2px`；不得用去掉 outline 代替设计焦点。
- hover/press：只调整背景、边框或轻微透明度，不使用缩放和弹跳。
- 图标：统一使用 Lucide 风格线性图标，常用 `16px` / `20px` / `24px`，stroke `1.75–2px`；图标按钮必须有可访问名称。
- 动效：hover `120ms`、Popover/Tooltip `180ms`、Dialog/Drawer `240ms`，统一 ease-out；不做页面级入场动画。
- `prefers-reduced-motion: reduce` 时关闭非必要动画，保留即时状态反馈。

## 12. 自适应布局

| Breakpoint  | Build-time Token | Value    |
| ----------- | ---------------- | -------- |
| Small       | `breakpoint-sm`  | `640px`  |
| Medium      | `breakpoint-md`  | `768px`  |
| Large       | `breakpoint-lg`  | `1024px` |
| Extra Large | `breakpoint-xl`  | `1280px` |
| 2X Large    | `breakpoint-2xl` | `1536px` |

断点使用构建时主题配置或共享常量，因为原生 CSS media query 不能可靠使用 CSS custom property。断点用于统一布局切换，但优先使用 Grid/Flex、`minmax()` 和容器查询实现内容驱动自适应。

1. 所有页面必须随可用宽度自适应，不以固定桌面宽度作为唯一布局。
2. 组件使用容器宽度和内容约束，避免散落设备型号判断。
3. 官网移动端可以使用顶部导航 + 汉堡菜单，但“进入应用”在窄屏仍应清晰可见。
4. 应用工作台移动端使用底部导航；卡片网格按空间自动减少列数，关键收益/风险指标不得因缩窄而拆散或隐藏。
5. 管理端宽屏保留侧边栏和高密度表格；窄屏折叠侧边栏、优先保留关键列，并通过横向滚动或详情视图完成必要操作。
6. 表单在窄屏改为单列；弹窗不得超出视口；图表和 DataTable 必须有可用的最小高度与溢出策略。
7. 不允许仅通过 `display:none` 隐藏关键风险、权限、状态或操作反馈来适配小屏。
8. 官网和应用工作台内容最大宽 `1280px`：小于 `768px` 水平 gutter `16px`，`768–1279px` 为 `24px`，`1280px+` 为 `32px`。
9. 管理端内容区流式填充；宽度低于 `1024px` 时侧边栏切换为抽屉，低于 `768px` 时表单默认单列。
10. 卡片网格默认：`<768px` 1 列、`768px+` 2 列、`1024px+` 3 列、`1280px+` 最多 4 列；内容不足时不得强行填满。
11. 应用工作台页面底部必须预留底部导航和安全区空间，避免主操作按钮被固定导航遮挡。

## 13. 风险与合规

以下位置必须展示标准风险披露：策略详情、信号详情、订阅前、创建模拟盘前、会员申请/开通确认和新增高风险确认流程。

> QuantFlow 不提供投资建议，不承诺任何收益。所有策略信号仅供参考，加密资产价格波动较大，用户需自行承担交易风险。历史表现不代表未来收益。

禁止金币、火箭、奖杯、赌场筹码、收益爆炸动效，以及“稳赚、保本、零风险”等表达。

## 14. 实现验收

1. 首次加载默认浅色，且无深色主题切换入口。
2. 页面主体由白/灰/黑组成，主按钮为深灰/黑，蓝色保持低频。
3. 官网符合轻量营销表达和“进入应用”主 CTA；应用工作台符合浅灰背景、白卡、轻边框、少阴影；管理端符合侧边栏、白内容区和高密度表格。
4. 数据语义色不被用作品牌营销色。
5. 收益和风险配对、模拟标识、风险披露与可访问性要求通过测试。
6. 样式使用语义 tokens；深色仅为未来覆盖结构，不增加 MVP 页面与视觉回归矩阵。
7. v0 颜色值与 token 名称一致，所有页面在窄屏、中等宽度和宽屏下信息完整、无不可访问溢出。
8. 应用工作台移动端主导航为固定底部导航栏，官网移动端可使用汉堡菜单，管理端移动端仅作为基础查看。
