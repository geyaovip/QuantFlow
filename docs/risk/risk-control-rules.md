## 1. 文档目标

本文档定义 QuantFlow MVP 风控规则，确保策略、信号、模拟盘、会员和管理后台在不进行真实交易的前提下仍具备风险提示、异常识别和处理能力。

QuantFlow 不提供投资建议，不承诺任何收益。所有策略信号仅供参考，加密资产价格波动较大，用户需自行承担交易风险。历史表现不代表未来收益。

## 2. 风控原则

1. 先保护用户，再展示策略。
2. 先模拟验证，再考虑真实交易。
3. MVP 不连接交易所、不读取真实资产、不真实下单。
4. 所有策略必须展示风险。
5. 所有收益数据必须展示配套风险指标。
6. 所有高风险策略必须有显著提示。
7. 所有异常行为必须记录日志。
8. 平台和策略作者不得承诺收益。

## 3. MVP 风控范围

MVP 风控覆盖：

1. 策略上线审核。
2. 策略风险评级。
3. 策略表现数据校验。
4. 信号有效期和异常状态。
5. 模拟盘仓位限制。
6. 模拟盘回撤和连续亏损提示。
7. 行情数据延迟或异常提示。
8. 会员申请或开通前风险提示。
9. 管理端风险事件处理。

MVP 不覆盖：

1. 真实交易所 API Key 风控。
2. 真实下单风控。
3. 半自动交易确认风控。
4. 全自动交易熔断。

以上能力仅在后续版本预留。

## 4. 策略风控

### 4.1 策略上线审核

策略上线前必须检查：

1. 是否存在收益承诺。
2. 是否展示最大回撤。
3. 是否展示最大连续亏损次数。
4. 是否说明适合行情。
5. 是否说明不适合行情。
6. 是否说明可能失败的场景。
7. 是否区分回测数据和模拟数据。
8. 是否存在数据样本不足。
9. 是否使用误导性策略名称。

### 4.2 风险等级

使用最近 30 天有效样本的最大回撤作为基础等级，并允许风控事件向上覆盖：

| 等级     | 30 日最大回撤 | 行为                                                          |
| -------- | ------------- | ------------------------------------------------------------- |
| low      | `<10%`        | 正常展示                                                      |
| medium   | `10%–<20%`    | 展示风险说明                                                  |
| high     | `20%–<30%`    | 强提示，订阅/创建模拟盘前确认                                 |
| critical | `>=30%`       | 自动进入 risk_watch，暂停新订阅、新信号和新模拟成交，人工复核 |

样本量 `<30` 笔为 insufficient，只能展示“样本不足”；`30–99` 笔为 provisional；`>=100` 笔才允许参与正式排行。风险等级不能因样本不足被包装成低风险。

### 4.3 禁止上线策略

禁止上线：

1. 宣传稳赚、保本、固定收益、零风险。
2. 不展示风险。
3. 不展示最大回撤。
4. 数据明显异常。
5. 无法解释策略逻辑。
6. 使用误导性命名。
7. 暗示 MVP 可真实下单或自动交易。

## 5. 信号风控

### 5.1 信号生成规则

信号必须包含：

1. 策略 ID。
2. 币种。
3. 方向。
4. 触发价格。
5. 当前价格快照。
6. 建议仓位。
7. 止损位置。
8. 止盈位置。
9. 有效期。
10. 触发原因。
11. 风险等级。

### 5.2 信号失效规则

信号在以下情况失效：

1. 超过有效期。
2. 策略暂停。
3. 策略进入风险观察。
4. 行情数据异常。
5. 管理员取消。
6. 当前价格相对触发价不利偏离超过 2%。

### 5.3 信号展示规则

1. 有效信号可查看和加入模拟盘。
2. 过期信号仅可查看历史，不可新建模拟执行。
3. 风控观察信号必须展示风险说明。
4. 信号详情必须展示“不承诺收益”免责声明。
5. MVP 不允许从信号直接真实下单。

## 6. 模拟盘风控

### 6.1 默认限制

1. 初始资金默认 10,000 USDT。
2. 单笔最大仓位默认 10%。
3. 最大持仓数量默认 3。
4. MVP 默认不使用杠杆。
5. MVP 杠杆固定 1 倍。
6. 默认开启止损。
7. 默认开启策略风控。

### 6.2 风控事件

模拟盘触发风险事件：

1. 当前权益回撤超过已配置且版本化的阈值。
2. 策略连续亏损。
3. 单笔亏损超过已配置且版本化的阈值。
4. 行情数据延迟。
5. 策略暂停。
6. 模拟收益异常。
7. 模拟计算失败。

具体阈值：

1. 模拟盘最大回撤达到 10%：warning，通知用户并记录事件。
2. 最大回撤达到 20%：high，自动暂停该模拟盘的新成交。
3. 最大回撤达到 30%：critical，暂停并升级管理端处理。
4. 连续亏损 3 笔：warning；连续亏损 5 笔：high，暂停新成交等待复核。
5. 单日绝对收益率超过 20%：标记 `performance_anomaly`，暂停排行与对外突出展示，复核行情和计算。
6. 行情快照年龄超过 120 秒：拒绝新成交；连续 5 分钟无有效数据：暂停受影响模拟盘。

### 6.3 处理规则

1. 低风险事件：记录并提示。
2. 中风险事件：提示用户并进入管理端风险列表。
3. 高风险事件：按 `risk-v1` 自动暂停新成交或限制新模拟，同时通知用户和管理员。
4. 严重风险事件：立即暂停、升级管理员处理；恢复必须填写原因并记录审计日志。
5. 自动动作必须幂等、记录规则版本、触发值和关联快照，并支持管理员安全恢复。

## 7. 内容与合规风控

### 7.1 必须展示免责声明

以下位置必须展示：

1. 首页底部。
2. 策略详情页。
3. 信号详情。
4. 订阅策略前。
5. 创建模拟盘前。
6. 会员申请/开通页。
7. 风险测评页。

标准文案：

QuantFlow 不提供投资建议，不承诺任何收益。所有策略信号仅供参考，加密资产价格波动较大，用户需自行承担交易风险。历史表现不代表未来收益。

### 7.2 禁止表达

禁止：

1. 稳赚。
2. 保本。
3. 固定收益。
4. 零风险。
5. 必涨。
6. 百分百胜率。
7. 躺赚。
8. 无脑赚钱。
9. 永不爆仓。
10. 稳赚不亏。

## 8. 管理端风险处理

### 8.1 风险事件字段

1. 风险 ID。
2. 风险类型。
3. 风险等级。
4. 关联用户。
5. 关联策略。
6. 关联信号。
7. 关联模拟盘。
8. 触发时间。
9. 处理状态。
10. 处理人。
11. 处理备注。

### 8.2 处理状态

1. 待处理。
2. 处理中。
3. 已处理。
4. 已忽略。
5. 已升级。

### 8.3 管理操作

1. 查看风险详情。
2. 暂停策略。
3. 取消信号。
4. 暂停模拟盘。
5. 标记已处理。
6. 添加处理备注。
7. 升级给高级管理员。

所有操作必须记录审计日志。

## 9. 后续版本预留

半自动交易阶段新增：

1. API 权限检测。
2. 交易金额限制。
3. 价格偏离检测。
4. 信号有效期检测。
5. 用户余额检测。
6. 仓位上限检测。
7. 杠杆上限检测。
8. 重复下单检测。

全自动交易阶段新增：

1. 自动交易总开关。
2. 用户级暂停。
3. 策略级暂停。
4. 平台级熔断。
5. 连续亏损暂停。
6. 单日亏损暂停。
7. 极端行情暂停。
8. 异常订单暂停。

以上能力不进入 MVP。

## 10. 验收标准

1. 风控规则覆盖策略、信号、模拟盘、内容、会员申请/开通和管理端处理。
2. 所有风险提示明确说明 QuantFlow 不承诺收益。
3. 所有收益展示均有最大回撤、连续亏损、风险等级或样本不足提示。
4. MVP 风控不依赖真实交易所 API 或真实订单。
5. 管理端风险事件具备状态流转和审计日志。
