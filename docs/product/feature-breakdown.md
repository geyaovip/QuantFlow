## 1. 文档目标

本文档将 QuantFlow PRD 拆解为可排期、可设计、可开发、可测试的功能模块。

QuantFlow MVP 是策略信号与模拟验证平台，不是实盘交易平台。产品采用“官网营销页 + 应用工作台 + 管理后台”分离架构：`/` 为官网，`/app` 为应用工作台，`/admin` 为管理后台。MVP 不包含真实交易所 API 连接、真实下单、半自动交易、全自动交易。所有与真实交易相关的能力仅保留架构扩展点，不进入用户可用功能。

## 2. 产品阶段边界

### 2.1 MVP 阶段

目标是跑通三个核心闭环：

1. 策略发现闭环：用户能发现、筛选、比较策略。
2. 信号订阅闭环：用户能查看策略信号、理解触发原因和风险。
3. 模拟验证闭环：用户能用模拟资金验证策略表现。

MVP 成功标准不是产生真实交易，而是用户愿意持续查看信号、创建模拟盘，并申请或使用更完整的会员权益。

### 2.2 后续阶段

半自动交易阶段仅在用户连接交易所 API 并手动确认后执行交易，MVP 不开放。

全自动交易阶段仅在风控、策略表现、安全机制和用户信任成熟后开放，MVP 不开放。

## 3. MVP 功能优先级

### 3.1 P0 必须上线

1. 用户邮箱验证码登录、首次登录建号和退出；邮件由 Resend 发送。
2. 官网首页 `/`。
3. 应用工作台 `/app`，必须重定向到 `/app/strategies`。
4. 策略广场。
5. 策略详情。
6. 信号中心。
7. 我的策略。
8. 模拟盘创建、列表和详情。
9. 会员计划、Plisio 支付、人工/邀请码/测试订阅生命周期和权益控制。
10. 个人中心。
11. 管理端 `/admin` 邮箱验证码登录、数据看板、用户管理、策略管理、信号管理、模拟盘管理、会员管理、风险管理和权限管理。
12. 基础风险提示和免责声明。

### 3.2 P1 可延后但建议预留

1. 风险测评。
2. 策略收藏。
3. 策略排行。
4. 高级筛选。
5. 邮件通知。
6. Telegram 通知。
7. 行情中心。
8. 后台内容管理。
9. 帮助中心。

### 3.3 P2 不进入 MVP

1. 真实交易所 API 连接。
2. 真实资产读取。
3. 半自动交易。
4. 全自动交易。
5. 策略作者端公开入驻。
6. KOL 分销、交易所返佣、App 端。

## 4. 官网模块说明

路径：`/`。

### 4.1 官网首页

模块说明：

1. 展示 QuantFlow 的产品定位、核心功能、策略信号、模拟盘、风控理念、定价和风险提示。
2. 首屏必须强调“先看信号、再模拟验证”，不得暗示用户可在 MVP 内连接交易所或自动下单。
3. 风险提示必须包含“不承诺收益、历史表现不代表未来收益、加密资产波动较大”。
4. 官网右上角提供“进入应用”按钮，作为主要 CTA。
5. 官网可以更轻量、更有营销表达，但不能出现稳赚、保本、躺赚、零风险等表述。

主要验收点：

1. 未登录用户可以浏览公开内容。
2. 点击“进入应用”进入 `/login?next=/app/strategies`，登录成功后进入 `/app/strategies`。
3. 任意收益指标旁边必须展示最大回撤、风险等级或样本不足提示。

## 5. 应用工作台模块说明

路径前缀：`/app`。

### 5.1 策略广场

模块说明：

1. 用户可按币种、策略类型、风险等级、收益周期、最大回撤、是否免费、是否支持模拟盘筛选策略。
2. 策略卡片必须展示收益表现与风险指标，包括近 7 日表现、近 30 日表现、最大回撤、风险等级、胜率和信号频率。
3. 高风险策略必须有明显但克制的风险提示。

主要验收点：

1. 筛选、排序和服务端编号分页可用；用户端默认每页 20 条。
2. 暂停、下架、风险观察策略不可订阅。
3. 不得出现“稳赚、保本、固定收益、零风险”等禁止表达。

### 5.2 策略详情

模块说明：

1. 解释策略逻辑、适合行情、不适合行情、当前信号、历史表现、收益曲线、回撤曲线、历史信号和风险说明。
2. 回测数据和模拟数据必须明确区分。
3. 数据样本不足时必须展示“数据样本较少，参考价值有限”。

主要验收点：

1. 策略必须有风险等级、最大回撤、最大连续亏损和失败场景说明。
2. 收益曲线旁边必须展示回撤曲线或回撤指标。
3. MVP 不展示半自动执行或开启自动交易按钮。

### 5.3 信号中心

模块说明：

1. 展示用户可访问的全部信号、买入信号、卖出信号、观望信号、有效信号、过期信号、已模拟信号。
2. 信号字段包括策略名称、币种、方向、触发价格、当前价格、建议仓位、止损、止盈、有效期、风险等级和状态。
3. MVP 仅支持查看、收藏、提醒和加入模拟盘，不支持真实下单。

主要验收点：

1. 用户无权限查看付费信号时展示升级引导。
2. 信号详情弹窗必须展示触发原因和风险提示。
3. 过期、取消、策略暂停和风控观察信号不可加入新的模拟执行。

### 5.4 模拟盘

模块说明：

1. 用户可选择策略、币种、初始模拟资金、单笔最大仓位、止损止盈规则创建模拟盘。
2. 默认配置为 10,000 USDT、单笔最大仓位 10%、最大持仓 3 个、不使用杠杆、开启止损和策略风控。
3. 模拟盘只记录模拟订单、模拟交易和模拟权益变化，不产生真实交易。

主要验收点：

1. 创建流程清楚展示这不是实盘交易。
2. 模拟盘详情必须展示当前权益、累计收益、最大回撤、历史交易、风控触发记录。
3. 免费用户数量限制与会员权益一致。

### 5.5 行情中心（P1）

模块说明：

1. 展示 BTC、ETH、SOL 等主流币行情数据。
2. 展示当前价格、24 小时涨跌幅、成交量、相关策略和最新信号。
3. 行情中心用于辅助理解策略和信号，不提供真实交易入口。

主要验收点：

1. 行情数据展示来源、更新时间和数据延迟状态。
2. 相关策略和信号入口跳转到 `/app` 内对应页面。
3. 不出现买入、下单、连接交易所等实盘操作。

行情中心不进入 `v1.0.0` 导航和 P0 验收；后续启用时复用 Market Data API，并单独补充页面验收。

### 5.6 会员订阅

模块说明：

1. 提供 Free、Pro、Premium 三档权益，价格和配额以 `membership-plans.md` 为准。
2. 权益差异围绕策略访问、实时信号、历史数据、模拟盘数量、通知渠道和策略风险分析。
3. 会员页面不得使用夸张收益承诺作为转化卖点。

主要验收点：

1. 超出权益时展示升级提示。
2. 申请或开通会员前展示风险免责声明。
3. Plisio、人工、邀请码和测试订阅状态可追踪；页面明确支付只开通会员容量。

## 6. 管理端模块说明

路径前缀：`/admin`。

### 6.1 数据看板

展示用户、策略、信号、模拟盘、商业和风险数据。运营数据允许 5-15 分钟延迟，但风险告警应尽量实时。

### 6.2 用户管理

支持用户列表、详情、会员状态、订阅记录、模拟盘记录、风险测评结果和操作日志。敏感操作必须记录管理员、时间、IP 和变更前后值。

### 6.3 策略管理

支持策略创建、编辑、审核、上线、暂停、风险观察和下架。策略上线前必须检查收益承诺、风险说明、最大回撤、适合行情、不适合行情和样本量。

### 6.4 信号管理

支持查看、取消、标记异常、重新推送和暂停相关策略。异常信号必须进入风险事件。

### 6.5 模拟盘管理

支持查看模拟盘、暂停、恢复、标记异常、查看交易记录和风控事件。不得将模拟交易误标识为真实交易。

### 6.6 会员管理

支持订阅分页列表、手动开通、延长和关闭会员，不提供订单和退款操作。所有变更必须记录审计日志。

### 6.7 风险管理

集中处理策略异常、信号异常、模拟盘异常、行情数据异常、用户投诉和系统告警。

### 6.8 内容管理（P1）

后台 CMS 不进入 `v1.0.0`。MVP 官网内容、风险提示和新手说明由版本控制中的代码/配置维护；系统公告继续使用既有管理能力。后续 CMS 不得出现收益承诺或禁止表达。

### 6.9 权限管理

支持角色、权限、管理员账号和操作日志。超级管理员权限应最小化分配。

## 7. 后续版本预留

只允许在路线或 ADR 中描述未来能力。MVP 不创建交易所密钥表、不注册实盘 API、不建立前端路由；四个禁用 feature flag 保留为服务端启动门禁。

## 8. 验收标准

1. 功能列表能区分 P0、P1、P2，且 P2 不进入 MVP。
2. 官网、应用工作台、管理端每个模块都有清晰目标、字段、状态和操作边界。
3. 所有收益指标配套风险指标。
4. 所有风险提示明确说明 QuantFlow 不承诺收益。
5. MVP 不存在真实交易所 API 连接、真实资产、真实下单、半自动交易和全自动交易入口。
6. `/`、`/app`、`/admin` 路径职责清晰，“进入应用”作为官网主要 CTA。
