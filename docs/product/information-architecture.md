## 1. 文档目标

本文档定义 QuantFlow MVP 的页面结构、导航层级、核心用户路径和权限状态，供产品、设计、前端和测试团队使用。

QuantFlow MVP 采用“官网营销页 + 应用工作台 + 管理后台”分离架构。根路径 `/` 进入官网首页，`/app` 进入应用工作台，`/admin` 进入独立管理后台。MVP 不展示真实交易所 API 连接、真实下单、半自动交易或全自动交易入口。

## 2. 信息架构原则

1. 先理解策略，再查看信号，再创建模拟盘。
2. 每个收益类入口必须同时可见风险信息。
3. 高风险信息不隐藏在二级页面，策略卡片和详情页均需展示。
4. 模拟盘与真实交易必须在命名、视觉、交互上严格区分。
5. 会员转化以数据完整度、模拟盘能力和通知能力为卖点，不以收益承诺为卖点。
6. 管理端优先保证审核、风控和审计闭环。
7. 官网负责品牌、信任、转化和风险教育；应用端负责具体功能使用；管理端负责运营管理。
8. 官网和应用端共用基础 Design Tokens，但布局和信息密度可以不同。
9. MVP 默认浅色主题；官网可更轻量、更有营销表达，应用端以浅灰画布和白色内容表面组织信息，管理端以侧边栏和高密度白色表格区组织任务。
10. 所有列表使用服务端编号分页：应用端默认 20 条，管理端默认 50 条；MVP 不使用无限滚动。

## 3. 官网站点地图

路径：`/`。

### 3.1 官网导航

1. 产品定位。
2. 核心功能。
3. 策略信号。
4. 模拟盘。
5. 风控理念。
6. 定价。
7. 风险提示。
8. “进入应用”。

移动端规则：

1. 官网移动端可以使用顶部导航 + 汉堡菜单。
2. “进入应用”必须保持清晰可见。
3. 汉堡菜单只承载官网锚点和辅助链接。

### 3.2 官网首页层级

1. 顶部导航。
2. 首屏主视觉。
3. 产品定位与核心主张。
4. 核心功能说明。
5. 策略信号预览。
6. 模拟盘说明。
7. 风控理念。
8. 使用流程。
9. 定价与会员权益说明。
10. 风险提示。
11. “进入应用” CTA。
12. 页脚。

模块说明：

官网首页必须在 10 秒内让用户理解 QuantFlow 是策略信号和模拟验证平台，不是承诺收益或直接代客交易的平台。官网可以更轻量、更有营销表达，但不得出现稳赚、保本、躺赚、零风险等表述。

### 3.3 官网 CTA 规则

1. “进入应用”是官网主要 CTA。
2. 未登录时点击进入 `/login?next=/app/strategies`，完成用户端邮箱验证码登录后进入 `/app/strategies`。
3. 已登录访问 `/app` 必须重定向到 `/app/strategies`；未登录访问 `/app/*` 必须重定向到 `/login?next=原路径`。
4. 官网不承载完整应用功能，不在官网内完成策略筛选、模拟盘管理或后台操作。

## 4. 应用工作台站点地图

路径前缀：`/app`。

### 4.1 应用端导航

桌面端导航：

1. 策略广场：`/app/strategies`。
2. 策略详情：`/app/strategies/:id`。
3. 信号中心：`/app/signals`。
4. 模拟盘：`/app/paper-trading`。
5. 我的策略：`/app/my-strategies`。
6. 会员订阅：`/app/membership`。
7. 个人中心：`/app/profile`。

移动端导航：

1. 应用端移动端必须使用底部导航栏作为主导航。
2. 不使用左上角汉堡菜单作为主导航。
3. 顶部区域用于当前页面标题、返回、通知、搜索等辅助操作。
4. 左上角不作为主功能入口，最多用于 Logo 或返回按钮。
5. 底部导航固定为四个核心入口：策略、信号、模拟盘、我的。
6. 底部导航必须固定、易点击、适配安全区，并避免遮挡页面主操作按钮。

### 4.2 策略广场层级

1. 策略搜索。
2. 策略分类筛选。
3. 风险等级筛选。
4. 收益周期筛选。
5. 最大回撤筛选。
6. 策略卡片列表。
7. 编号分页，默认每页 20 条；MVP 不使用无限滚动。
8. 风险提示固定区域。

模块说明：

策略卡片必须展示策略名称、类型、支持币种、适合行情、风险等级、近 7 日表现、近 30 日表现、最大回撤、胜率、信号频率、当前信号和模拟盘支持状态。

### 4.3 策略详情层级

1. 策略头部信息。
2. 当前信号。
3. 策略表现概览。
4. 收益曲线与回撤曲线。
5. 历史信号。
6. 策略逻辑说明。
7. 适合行情与不适合行情。
8. 风险说明。
9. 模拟盘入口。
10. 订阅入口。
11. 风险提示。

模块说明：

策略详情页是用户决策页，必须帮助用户理解策略为什么可能有效，以及在哪些情况下可能失效。

### 4.4 信号中心层级

1. 信号分类标签。
2. 筛选和排序。
3. 信号列表。
4. 信号详情弹窗。
5. 加入模拟盘。
6. 收藏或提醒（P1）。

模块说明：

MVP 信号中心不允许出现真实下单、确认交易或开启自动化按钮。信号只能用于查看和模拟验证；收藏与提醒作为 P1 能力预留。

### 4.5 模拟盘层级

1. 模拟盘列表。
2. 创建模拟盘。
3. 模拟盘详情。
4. 模拟交易记录。
5. 模拟持仓。
6. 收益曲线与回撤曲线。
7. 风控触发记录。
8. 运行参数。

模块说明：

所有模拟盘页面必须明确标识“模拟盘”，避免用户误解为真实交易。

### 4.6 行情中心（P1）

1. 主流币行情列表。
2. 当前价格。
3. 24 小时涨跌幅。
4. 成交量。
5. 相关策略。
6. 最新信号。
7. 风险提示。

模块说明：

行情中心不进入 `v1.0.0` 路由和导航。后续启用时用于辅助用户理解策略和信号，不提供真实交易入口。

### 4.7 会员订阅层级

1. 会员权益对比。
2. Free、Plus、Pro 计划。
3. 权益说明。
4. 风险免责声明。
5. 内测资格/邀请码说明。

模块说明：

订阅页只能承诺功能权益，不得承诺策略收益或交易结果。

## 5. 管理端站点地图

路径前缀：`/admin`。

### 5.1 一级菜单

管理端一级菜单固定在侧边栏，主内容区保持白色、高密度和任务导向；不使用大面积品牌色区分模块。

管理端以桌面端为主，但移动端仍须支持权限允许的必要查看和管理操作，不提供独立底部导航。

1. 数据看板。
2. 用户管理。
3. 策略管理。
4. 信号管理。
5. 模拟盘管理。
6. 会员管理。
7. 风险管理。
8. 通知管理。
9. 系统日志。
10. 权限管理。

### 5.2 管理端页面说明

数据看板展示平台运行概况，同时包含收益和风险数据。

用户管理处理用户状态、会员、订阅、模拟盘、风险测评和操作记录。

策略管理处理策略创建、审核、上线、暂停、风险观察和下架。

信号管理处理信号状态、异常标记、取消和重新推送。

模拟盘管理处理模拟盘运行状态、异常模拟结果和风控事件。

会员管理处理会员计划、人工/邀请码/测试订阅、Plisio 支付核查和手动调整，不处理退款。

风险管理处理策略异常、信号异常、模拟盘异常、用户投诉、行情数据异常和系统告警。

权限管理处理管理员、角色、权限和审计日志。

## 6. 核心用户路径

### 6.1 新用户路径

1. 访问根路径 `/`，进入官网首页。
2. 理解 QuantFlow 定位、核心功能、风控理念和风险提示。
3. 点击“进入应用”进入 `/login?next=/app/strategies`。
4. 登录成功后在 `/app/strategies` 查看策略广场。
5. 进入策略详情。
6. 通过 Resend 邮件验证码登录；首次成功验证时创建用户账号。
7. 创建模拟盘。
8. 查看模拟盘表现。
9. 根据权益提示考虑会员订阅。

### 6.2 已登录免费用户路径

1. 进入 `/app/strategies` 查看免费策略。
2. 查看延迟或有限信号。
3. 按配置的 Free 权益创建模拟盘。
4. 查看基础收益和回撤。
5. 触达权益限制。
6. 查看会员计划。

### 6.3 会员用户路径

1. 在 `/app` 内查看更多策略。
2. 接收实时信号。
3. 按当前计划配额创建模拟盘。
4. 查看完整历史信号和表现。
5. 设置通知提醒。
6. 持续观察收益与风险。

### 6.4 管理员路径

1. 访问 `/admin`。
2. 使用已授权管理员邮箱接收 Resend 验证码并登录管理端。
3. 查看风险和运行数据。
4. 审核策略。
5. 监控信号。
6. 处理异常模拟盘。
7. 处理人工/邀请码订阅和会员配额。
8. 查看操作日志。

## 7. 页面权限与状态

### 7.1 用户状态

1. 未登录：可看官网首页、公开策略、部分延迟信号。
2. Free：可看免费策略和有限历史数据，并按计划配置创建模拟盘。
3. Plus：可看大部分策略、实时信号、多个模拟盘、完整历史数据。
4. Pro：可看全部高级策略、更多模拟盘、深度策略报告。
5. 禁用用户：不可登录或访问受保护功能。

### 7.2 策略状态

1. 草稿：仅管理端可见。
2. 待审核：仅管理端可见。
3. 已上线：按访问权益在用户端可见。
4. 已暂停：用户端可见但不可订阅或创建新模拟盘。
5. 风险观察：用户端展示风险提示并限制操作。
6. 已下架：用户端不可见。

### 7.3 信号状态

1. 有效。
2. 已过期。
3. 已取消。
4. 策略暂停。
5. 风控阻断。

“已模拟”“已收藏”“已提醒”属于当前用户与信号的关系状态，不属于信号生命周期。

## 8. 验收标准

1. 官网 `/`、应用工作台 `/app` 和管理后台 `/admin` 站点地图覆盖 MVP 所有 P0 页面。
2. 页面路径中不存在真实交易所 API 连接、真实下单、半自动交易或全自动交易入口。
3. 每个收益展示页面都有风险、回撤或免责声明入口。
4. 未登录、Free、Plus、Pro 和管理员权限状态清晰。
5. 策略、信号、模拟盘状态可被前端、后端和测试团队直接转化为状态机。
6. 官网“进入应用”CTA 先进入用户端登录页，登录成功后进入 `/app/strategies`；访问 `/app` 必须重定向到该路径。
