## 1. 文档目标

本文档定义 QuantFlow MVP 的用户端权益权限、管理端 RBAC 权限和敏感操作审计规则。

QuantFlow MVP 不开放真实交易所 API 连接、真实下单、半自动交易和全自动交易权限。相关权限仅作为后续版本预留，不在 MVP 分配给任何角色。

用户端和管理端统一使用 Resend 投递的邮箱验证码登录，但身份空间和会话 audience 分离。管理员必须先存在于 `admin_users` 且状态启用，验证任意邮箱不得自动创建管理员。完整规则见 `authentication.md`。

## 2. 权限设计原则

1. 最小权限：每个角色只拥有完成职责所需的权限。
2. 前后端双校验：前端隐藏入口，后端强制校验。
3. 权益与权限分离：会员等级决定用户端权益，RBAC 决定管理端操作。
4. 敏感操作可审计：所有策略、信号、会员、风险和权限变更必须记录日志。
5. 真实交易权限默认不存在：MVP 不创建可用真实交易权限。

## 3. 用户端角色

### 3.1 Anonymous

权限：

1. 查看首页。
2. 查看公开策略列表。
3. 查看部分公开策略信息。
4. 查看延迟或脱敏信号预览。
5. 查看会员计划。

限制：

1. 不可订阅策略。
2. 不可查看完整信号。
3. 不可创建模拟盘。
4. 不可访问个人中心。

### 3.2 Free

权限：

1. 查看免费策略。
2. 查看有限信号。
3. 最多创建 1 个活跃模拟盘并订阅 3 个活跃策略。
4. 查看基础行情。
5. 接收站内通知。
6. 查看有限历史数据。

限制：

1. 不可查看高级策略完整数据。
2. 不可超过 1 个活跃模拟盘、3 个活跃策略订阅和 30 天历史范围。
3. 不可查看完整历史信号。

### 3.3 Pro

权限：

1. 查看大部分策略。
2. 接收实时信号。
3. 最多创建 10 个活跃模拟盘并订阅 20 个活跃策略。
4. 查看完整策略表现。
5. 查看完整历史信号。
6. 使用高级筛选。
7. 查看策略风险分析。

### 3.4 Premium

权限：

1. 查看全部高级策略。
2. 最多创建 30 个活跃模拟盘并订阅 50 个活跃策略。
3. 查看深度策略报告。
4. 使用更多通知渠道。
5. 使用更多数据分析。
6. 后续功能优先体验资格。

说明：

Premium 在 MVP 中不代表可连接真实交易所、不代表可真实下单、不代表可开启半自动或全自动交易。

## 4. 管理端角色

### 4.1 Super Admin

权限：

1. 所有管理端权限。
2. 管理管理员账号和角色。
3. 查看系统日志。
4. 处理严重风险事件。

约束：

1. 账号数量应严格限制。
2. 权限变更必须审计。

### 4.2 Operations Admin

权限：

1. 查看数据看板。
2. 管理策略内容。
3. 管理信号展示。
4. 管理系统公告。
5. 查看基础用户信息。

限制：

1. 不可修改管理员权限。
2. 不可查看敏感安全信息。

### 4.3 Risk Admin

权限：

1. 审核策略风险。
2. 暂停策略。
3. 取消异常信号。
4. 暂停异常模拟盘。
5. 处理风险事件。

### 4.4 Support Admin

权限：

1. 查看用户详情。
2. 查看会员状态。
3. 查看用户订阅和模拟盘。
4. 添加客服备注。

限制：

1. 不可手动开通会员。
2. 不可暂停策略。

### 4.5 Readonly Admin

权限：

1. 查看数据。
2. 查看日志。

限制：

1. 不可新增、编辑、删除或处理任何业务对象。

## 5. 权限矩阵

数据看板：

1. Super Admin：全部。
2. Operations Admin：查看。
3. Risk Admin：查看。
4. Support Admin：查看。
5. Readonly Admin：查看。

用户管理：

1. Super Admin：全部。
2. Operations Admin：查看基础信息。
3. Risk Admin：查看风险相关信息。
4. Support Admin：查看与添加备注。
5. Readonly Admin：查看。

策略管理：

1. Super Admin：全部。
2. Operations Admin：编辑与提交审核。
3. Risk Admin：审核与暂停。
4. Support Admin：无。
5. Readonly Admin：查看。

信号管理：

1. Super Admin：全部。
2. Operations Admin：查看与重新推送。
3. Risk Admin：取消与标记异常。
4. Support Admin：无。
5. Readonly Admin：查看。

模拟盘管理：

1. Super Admin：全部。
2. Operations Admin：查看。
3. Risk Admin：暂停与标记异常。
4. Support Admin：查看。
5. Readonly Admin：查看。

会员管理：

1. Super Admin：全部。
2. Operations Admin：查看。
3. Risk Admin：无。
4. Support Admin：查看。
5. Readonly Admin：查看。

风险管理：

1. Super Admin：全部。
2. Operations Admin：查看。
3. Risk Admin：处理。
4. Support Admin：查看。
5. Readonly Admin：查看。

权限管理：

1. Super Admin：全部。
2. 其他角色：无。

系统日志：

1. Super Admin：全部。
2. Operations Admin：查看业务日志。
3. Risk Admin：查看风险日志。
4. Support Admin：查看用户相关日志。
5. Readonly Admin：查看。

## 6. 敏感操作审计

必须审计：

1. 管理员登录。
2. 管理员权限变更。
3. 用户状态变更。
4. 会员手动开通、延长、关闭。
5. 策略审核、上线、暂停、下架。
6. 信号取消、异常标记、重新推送。
7. 模拟盘暂停、恢复、异常标记。
8. 风险事件处理。

审计字段：

1. 操作人。
2. 操作对象。
3. 操作类型。
4. 变更前数据。
5. 变更后数据。
6. IP。
7. User-Agent。
8. 操作时间。

## 7. 后续版本预留权限

MVP 权限种子、枚举和数据库中不得创建实盘、交易所连接、交易确认或自动化权限。未来阶段启动时重新进行威胁建模和最小权限设计，不复用模拟盘权限作为实盘授权。

## 8. 验收标准

1. 用户端 Anonymous、Free、Pro、Premium 权益边界清晰。
2. 管理端 RBAC 覆盖运营、风控、客服、只读和超级管理员；MVP 不创建财务角色。
3. 所有敏感操作均有审计日志。
4. 前端隐藏权限不足入口，后端强制校验权限。
5. MVP 中不存在任何可用的真实交易、半自动交易或全自动交易权限。
